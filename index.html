458. 可怜的小猪

有 buckets 桶液体，其中 正好有一桶 含有毒药，其余装的都是水。它们从外观看起来都一样。为了弄清楚哪只水桶含有毒药，你可以喂一些猪喝，通过观察猪是否会死进行判断。不幸的是，你只有 minutesToTest 分钟时间来确定哪桶液体是有毒的

喂猪的规则如下：

选择若干活猪进行喂养
可以允许小猪同时饮用任意数量的桶中的水，并且该过程不需要时间。
小猪喝完水后，必须有 minutesToDie 分钟的冷却时间。在这段时间里，你只能观察，而不允许继续喂猪。
过了 minutesToDie 分钟后，所有喝到毒药的猪都会死去，其他所有猪都会活下来。
重复这一过程，直到时间用完。
给你桶的数目 buckets ，minutesToDie 和 minutesToTest ，返回 在规定时间内判断哪个桶有毒所需的 最小 猪数 。

思路分析

这里设实验次数为k = [minutesToTest / minutesToDie], N = buckets

  1. 对于k==1的情况, 是一道耳熟能详的题目, 答案为$log_2(N - 1) + 1$

  2. 再无思路

  3. 反转题目, 考虑给到m头猪, 可以验证多少桶水

    • 首先$2^m$显然可以
    • 考虑到这里, 其实问题就变成了, 给一个m, 哪种运算可以使得结果最大
    • 个人得出结论为m!
    • 因为考虑到猪会死亡的情况, 未能想到$n^n$的情况

  4. 查看题解, 发现信息论相关, 豁然开朗

  5. 根据香农理论, 可以得出$n^n$的答案

关于信息论和香农

其基本理论为, 一个事件可以提供的信息量Q = - $log_2{P}$, P为事件发生的概率

通俗的解释, 我们观察到太阳从东边升起来, 这件事告诉我们的信息几乎没有, 因为这是很正常的事情. 或者这是必然的事情

再比如, 我们观察到飞机失事了, 那么我们可以得出更多的信息, 比如飞行员问题, 飞机硬件问题等等

这个问题较为抽象..不理解正常

非对称加密和RSA算法

对称加密

客户端A和服务端B通信, 设^为加密运算, v为解密运算

  1. 约定密钥K, K可对明文加密, 密文解密
  2. A发送明文S, 则密文为str = S ^ K, 发送str
  3. B收到str, 解密得明文S = str v K
  4. 反之, 亦可

其问题在于, 密钥K得传输时存在泄露的可能

非对称加密

客户端A和服务端B通信

  1. 设K, P为公钥和私钥, K加密, P解密
  2. B暴露出K, A获取K
  3. A发送明文S, 则密文为str = S ^ K, 发送str
  4. B收到str, 解密得明文S = str v P
  5. 至此实现了单方面得A发送, B接收的数据传输

其原理基于, 不能在NP时间内通过K解出P

由于计算复杂, 一般会通过一次非对称加密来传递一个对称加密的密钥, 后续通信使用对称加密即可

RSA算法

基于大数分解定理的一种非对称加密算法

大数分解, 一种陷门函数, 即可以在NP时间复杂度内正向计算f(x) = y, 但根据y解出x是NP完全问题(目前无法证明不是)

​ 密文 = $明文^E$ mod N

​ 明文= $密文^D$ mod N

生产过程:

  1. N = p * q

  2. L = lcm（p－1，q－1）

  3. 1 < E < L

    gcd（E，L）=1

  4. 1 < D < L

    E＊D mod L ＝ 1

简述

Knuth-Morris-Pratt 字符串查找算法，简称为 KMP算法，常用于在一个文本串 S 内查找一个模式串 P 的出现位置。

子串匹配

对于模式串 A，在文本串B中找出与A完全相同的子串的位置

例如：

A = “ab”

B = “aaaab”

枚举每一个i in B.index，对于每一个i，枚举j in A.index，check A[j] == B[i + j]，当一次j的循环完成跑完，check全部成立，则有ans_index = i

对于上述例子，上述算法的冗余操作在于，每次移动了一个单位，以下述例子进行解释

  1. abcabd 和 abcabcabd，第一次循环，会在i = 0，j = 5的时候终止， 因为abcabd和abcabc的最后一个不匹配

  2. 上述算法的下一步操作为，i++，j = 0，继续循环

  3. 这时候我们考虑i的位置，显然，i = 1,2都属于冗余操作，因为B[1] != A[0], B[2] != A[0]，接着看i = 3的情况，可以发现，i = 3有可能完成匹配，因为至少B[3] == A[0]

  4. 此时，我们再考虑j的位置，注意，这里有B[3] == A[0], B[4] == A[1], 所以我们j可以直接取2，即j = 2

  5. 显然直接取得i = 3, j = 2对于原先算法是一种优化，如果在O(1)的时间内获取这个信息，那么我们的优化即可完成

  6. 在此例中，在j = 5终止匹配，可以给我们一个信息，此信息为，A串的0-4与B串的0-4完全相同，我们需要处理出上述3、4两点的信息，需要用到A串的0-4和B串的0-4，又因为A串的0-4与B串的0-4完全相同，所以我们需要用的A串的0-4与A串的0-4，且注意，我们用A串去匹配B串，所以A串是重复使用的，因此我们可以预处理出A串的0-(A.length-1)的3、4两点提到的信息，实现O(1)得获得此信息

  7. 至此，优化完成，此为KMP算法，时间复杂度为O(M+N), M、N分别为A、B串的长度

存疑的点在于j = next[j] 导致了j最坏的情况下减小到0，所以时间复杂度为O(NM)

但是，KMP算法中i是递增的，不增的情况为失配时，

所以，KMP 的比对次数是 (n + 失配次数）
