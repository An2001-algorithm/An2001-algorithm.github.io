开发的遇到的bug

设置lottie播放流程监听导致的bug

用户反馈: 动画播放完成之后, 不能响应任何操作

前置条件: 产品需求为动画播放完成之后才能响应用户操作

解决流程:

  1. 查看用户反馈日志和截图, 显示用户在动画播放的界面显示之后没有任何操作

  2. 推断此时app不能响应用户操作

  3. 查看设备为huawei ELE-AL00 Android 10

  4. 尝试复现, 未果

  5. 查看bugly, 该版本此界面确有一个ANR上报, 推断是该ANR导致

  6. 修复掉该ANR

  8. 更新版本之后, 仍有用户反馈

  9. 重新查看日志, 发现没有用户操作40s后, 又打印了一条日志, 断定不是ANR导致

  10. 园区饭后散步, 突发奇想是点击事件设定代码未执行, 进而怀疑是lottie监听播放流程的问题

  11. 回归代码, 对于监听播放完成, 采用的方式是animatedFraction >= 0.999999 查看源码

  12.  /**
    * Implementors of this interface can add themselves as update listeners
    * to an <code>ValueAnimator</code> instance to receive callbacks on every  	 	  	  * animation frame, after the current frame's values have been calculated for 
    * that <code>ValueAnimator</code>.
    */
    ```

  13. 存在最后一帧小于0.999999 且播放完之后未进行回调的可能, 如此导致setOnClickListener未执行

  14. 改用addAnimatorListener(AnimatorListener) 修复掉该问题
